# Machine-Specific Configuration & Team Setup Guide

This document lists every part of the repo that is tied to a specific machine and explains what each team member must do to get the project running on their own computer.

---

## What Is Machine-Specific

### 1. `demo/.env` — API Keys (not committed to git)

This file must be created by each team member. It is listed in `.gitignore` and will never appear in the repo.

```
# demo/.env
AZURE_OPENAI_API_KEY=<your key>
TAVILY_API_KEY=<your key>

# Only needed if your PostgreSQL setup requires a username/password
# DATABASE_URL=postgresql://<user>:<password>@localhost:5432/procurement_agent
```

| Variable | Where to get it |
|---|---|
| `AZURE_OPENAI_API_KEY` | Azure Portal → your OpenAI resource → Keys and Endpoints |
| `TAVILY_API_KEY` | tavily.com → Dashboard → API Keys (free tier works) |
| `DATABASE_URL` | Only needed if your Postgres uses a non-default user or password |

---

### 2. `demo/config.py` — Hardcoded Azure Settings

The Azure endpoint and deployment name are **hardcoded** in `config.py`, not read from `.env`. If your Azure account uses a different endpoint or deployment name, you need to edit this file directly:

```python
# demo/config.py
AZURE_ENDPOINT   = "https://gw-sb-aoai-01.openai.azure.com/"  # ← change if different
AZURE_DEPLOYMENT = "gpt-5-mini"                                 # ← change if different
AZURE_API_VERSION = "2024-12-01-preview"                        # ← usually fine as-is
```

> **Note:** The API key itself (`AZURE_OPENAI_API_KEY`) is correctly read from `.env`. Only the endpoint URL and deployment name are hardcoded.

---

### 3. PostgreSQL — Local Database

The app connects to a local PostgreSQL database named `procurement_agent`. The default connection string (`postgresql://localhost:5432/procurement_agent`) assumes:
- PostgreSQL is running locally
- No username or password is required (peer/trust auth)

**If your Postgres requires a username or password**, set `DATABASE_URL` in `demo/.env`:
```
DATABASE_URL=postgresql://youruser:yourpassword@localhost:5432/procurement_agent
```

The database must also be populated from scratch — it is not included in the repo. See the full setup steps below.

---

### 4. `venv/` directories — Not committed

Both virtual environments are excluded from git:
- `venv/` (project root) — used for forecasting, inventory, and optimization
- `demo/venv/` — used for the Streamlit demo

Each team member must create these locally. See setup steps below.

---

### 5. `ui_issues/` folder — Runtime assets required by the app

The Streamlit app loads three PNG files from `ui_issues/` at startup:

| File | Used as |
|---|---|
| `ui_issues/logo_python.png` | Site favicon + sidebar logo + landing page logo |
| `ui_issues/user.png` | User avatar in chat messages |
| `ui_issues/cpu.png` | Assistant avatar in chat messages |

These **are committed to git** and will be present after cloning. If any are missing the app falls back to text defaults gracefully — but the icons will not appear.

---

### 6. `artifacts/forecasting/` — Generated outputs

The model evaluation artifacts (plots, metrics JSON, CSV predictions) in `artifacts/forecasting/` are generated by running the forecasting pipeline. Some may be committed for reference but should be regenerated locally to reflect the current model state.

## Full Setup Steps for a New Team Member

### Prerequisites

| Requirement | Notes |
|---|---|
| Python 3.11+ | 3.12 or 3.13 recommended |
| PostgreSQL 14+ | Must be running locally |
| Git | To clone the repo |
| Azure OpenAI API key | Shared by the team or individually provisioned |
| Tavily API key | Free tier at tavily.com |

---

### Step 1 — Clone the repo

```bash
git clone <repo-url>
cd procurement_agent
```

---

### Step 2 — Create and load the database

```bash
# Create the database (Linux)
sudo -u postgres psql -c "CREATE DATABASE procurement_agent;"

# macOS (if using Homebrew Postgres)
createdb procurement_agent

# Load schema and data — order is mandatory
sudo -u postgres psql -d procurement_agent -f sql/dimensions.sql
sudo -u postgres psql -d procurement_agent -f sql/facts.sql
sudo -u postgres psql -d procurement_agent -f sql/load/stage.sql
sudo -u postgres psql -d procurement_agent -f sql/load/copy_staging.sql
sudo -u postgres psql -d procurement_agent -f sql/load/load_dimensions.sql
sudo -u postgres psql -d procurement_agent -f sql/load/load_facts.sql
sudo -u postgres psql -d procurement_agent -f sql/load/load_bom.sql
sudo -u postgres psql -d procurement_agent -f sql/views.sql
```

Verify:
```bash
sudo -u postgres psql -d procurement_agent -c "SELECT COUNT(*) FROM vw_supplier_complete_profile;"
# Expected: 89
```

---

### Step 3 — Root Python environment

```bash
# From project root
python -m venv venv
source venv/bin/activate          # Windows: venv\Scripts\activate
pip install -r requirements.txt
```

---

### Step 4 — Run the forecasting and inventory pipeline

```bash
# From project root, with root venv active
python -m forecasting.run_production    # trains model + writes forecast to DB
python -m inventory.run_inventory       # builds inventory state + procurement requirements
```

These populate the tables the LP optimizer and demo agents query. They must be run at least once before launching the UI.

---

### Step 5 — Demo environment

```bash
cd demo
python -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

---

### Step 6 — Create `demo/.env`

```bash
# demo/.env  (create this file — it is gitignored)
AZURE_OPENAI_API_KEY=your-key-here
TAVILY_API_KEY=your-key-here
```

If your Postgres needs credentials:
```
DATABASE_URL=postgresql://youruser:yourpassword@localhost:5432/procurement_agent
```

---

### Step 7 — Check `demo/config.py`

Open `demo/config.py` and confirm:
- `AZURE_ENDPOINT` matches your Azure OpenAI resource URL
- `AZURE_DEPLOYMENT` matches the deployment name in your Azure resource

If they differ, edit the file directly.

---

### Step 8 — Verify everything works

```bash
# From demo/ with demo venv active
python -c "import psycopg2; conn = psycopg2.connect('postgresql://localhost:5432/procurement_agent'); print('DB OK'); conn.close()"
python -c "from llm import get_llm; llm = get_llm(); print(llm.invoke('Say hello in one word').content)"
python -c "from graph.builder import build_graph; app = build_graph(); print('Graph OK')"
python -c "from tools.pipeline_queries import DIRECT_PIPELINE_TOOLS; print(f'{len(DIRECT_PIPELINE_TOOLS)} tools loaded')"
```

---

### Step 9 — Launch

```bash
cd demo
source venv/bin/activate
streamlit run streamlit_app.py
# → http://localhost:8501
```

## Summary: What Each Person Needs to Do

| Task | One-time? | Notes |
|---|---|---|
| Create `demo/.env` | Once | Never committed — each person needs their own |
| Check `demo/config.py` Azure settings | Once | Edit if your Azure endpoint/deployment differs |
| Create and load PostgreSQL database | Once | Run 8 SQL files in order |
| Create root `venv/` | Once | `pip install -r requirements.txt` from project root |
| Run forecasting + inventory pipeline | Once (then as needed) | Must run before demo works |
| Create `demo/venv/` | Once | `pip install -r requirements.txt` from `demo/` |
| Confirm `ui_issues/` PNGs are present | Once | Should be present after `git clone` |
